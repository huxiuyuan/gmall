# Elasticsearch

> Elasticsearch：搜索引擎、NoSql(非关系型数据库)

## 搜索引擎

- 根据用户的需求结合特定的算法，运用特定策略检索出指定信息并反馈给用户的一门检索技术

## 相关概念

| Elasticsearch  | MySql       |
| -------------- | ----------- |
| indices 索引库 | database 库 |
| type 类型      | table 表    |
| document 文档  | row 行      |
| field 字段     | cloums 列   |
| mapping 映射   | schema 约束 |

## 基本操作

### 1.索引库操作

```
# 查询所有索引库
GET /_cat/indices?v

# 创建一个索引库
PUT /atguigu
{
  "settings": {
    # 分片数量
    "number_of_shards": 3,
    # 副本数量
    "number_of_replicas": 2
  }
}

# 查看单个索引库
GET /atguigu

# 删除索引库
DELETE /atguigu
```

### 2.映射操作

```
# 指定索引库声明映射
# 字符串类型：text(会进行分词)、keyword
PUT /atguigu/_mapping
{
  "propertis": {
    # 标题
    "title": {
      # 类型
      "type": "text",
      # 是否创建索引, 默认为true, 取决于要不要以该字段进行搜索
      "index": true,
      # 分词器
      "analyzer": "ik_max_word"
    },
    # 图片
    "images": {
      "type": "keyword",
      "index": false
    },
    # 价格
    "price": {
        "type": "double",
        "index": true
      }
    }
  }
}

PUT /atguigu/_mapping
{
  "properties": {
    "title": {
      "type": "text",
      "analyzer": "ik_max_word"
    },
    "images": {
      "type": "keyword",
      "index": false
    },
    "price": {
      "type": "double"
    }
  }
}

# 查看指定索引库的映射
GET /atguigu/_mapping

# 无法删除映射，只能删除整个索引库
```

### 3.文档CRUD

```
### 文档CRUD
## 新增：7.0版本后type废除，默认_doc
POST /atguigu/_doc/1
{
  "title": "小米手机",
  "images": "http://xiaomi.com/xm.jpg",
  "price": 1999
}

## 更新
# 覆盖更新
POST /atguigu/_doc/1
{
  "title": "小米手机",
  "images": "http://xiaomi.com/xm.jpg",
  "price": 2999
}
# 更新后titile、images为空
POST /atguigu/_doc/1
{
  "price": 2999
}


## 删除
DELETE /atguigu/_doc/1


## 查询
GET /atguigu/_search
```

## 使用Kibana在Elasticsearch中对数据进行分析

```
GET /_cat/indices?v

# 默认使用标准分词器
GET /_analyze
{
  "text": "我是中国人"
}

# 标准分词器
GET /_analyze
{
  "text": "我是中国人",
  "analyzer": "standard"
}

# ik_smart标准分词器
GET /_analyze
{
  "text": "我是中国人",
  "analyzer": "ik_smart"
}

# ik_max_word标准分词器
GET /_analyze
{
  "text": "我是中国人",
  "analyzer": "ik_max_word"
}
```
